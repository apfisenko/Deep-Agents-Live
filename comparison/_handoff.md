- **Что сравниваем между ранами:**
  - **Context Recall / Context Precision** (RAGAS) — меряют именно ретривал и **НЕ требуют стабильных

  - **End-to-end latency агента** (не latency вызова поиска, а полный трейс агента) — Langfuse снимает
    её с трейсов **автоматически**, в датасет класть не надо. Сравниваем агрегаты двух ранов. Это и
    ответ на вопрос «мерим ли / можем ли / должны ли датасетом»: **можем (бесплатно, из трейсов),
    меряем при прогоне, сравнением ранов FAISS↔Qdrant**.
  - **Answer-correctness** — как есть; может сдвинуться слабо или никак (лёгкие примеры) — нормально
    и проговорено.
  - (опц.) **число вызовов агента** — лучше ретривал → меньше итераций → быстрее/качественнее.
- **Ожидание честное:** в идеале context-метрики ↑ и/или latency меняется; возможно, по correctness
  не сдвинется ничего. Допустимо — демонстрируем механику before→after, не гарантированный «вин».

## 3. Ключевые решения ЭТОЙ сессии (которых ещё нет в plan.md decision log)

1. **Эмбеддинги — через OpenRouter API**, модель **`intfloat/multilingual-e5-large`** (1024d,
   та же, что в уроке §4.2). Почему ушли от локального fastembed:
   - локальная модель (1.5 ГБ в RAM) + флот параллельных python = **OOM на ноуте автора**;
   - полный `model.onnx` mxbai **не докачивался через прокси автора** (рвался; fp16 не грузится
     на CPU, int8 — медленный 20 t/s);
   - API: low-RAM (нет модели в ядре), быстро (~42 doc/s), $0.01/1M токенов. **Это убрало OOM.**
   - Ключ: `OPENROUTER_API_KEY` (в каноническом `.env` репо; `corpus.py` читает из env/.env).
2. **Корпус — публичный DBpedia-14** (`fancyzhx/dbpedia_14`): короткие аннотации, 14 категорий
   (метка класса = метаданные для демо фильтрации). Подменяется в `corpus.load_texts()`.
3. **Правда-2026 (ресёрч свежих API) изменила нарратив фильтрации/гибрида:**
   - **Chroma теперь pre-filter** → старый дефект «теряет 3 из 10» в pip-версии **не воспроизводится**
     (исторический — да, об этом урок).
   - **pgvector по дефолту под-возвращает** на ANN-индексе, НО только когда планировщик идёт
     через HNSW. На малой таблице он делает seq-scan и отдаёт честные K. Чтобы показать дефект
     надёжно — **`SET enable_seqscan = off`** + фильтр-категория, ортогональная запросу.
   - **Chroma hybrid/BM25 — Cloud-only**, в pip нет → гибрид демонстрируем на Qdrant; у pgvector
     гибрид = FTS+RRF руками. Подаём честно.
   - Вывод секции стал сильнее: «потеря результатов» — свойство механики, не ярлык вендора.

## 5. Что подтвердил прогон @5k (реальные числа)

- §1: топ-5 у трёх баз **идентичны** (взаимозаменяемы на простом поиске).

## 6. Грабли и уроки (ОБЯЗАТЕЛЬНО прочитать перед продолжением)

- **Память/OOM.** Корень был — **флот параллельных python** (докачка модели + бенчи + прогон
  одновременно, каждый ~1.5 ГБ). Правила: **один тяжёлый процесс за раз**; мониторить
  `free -m` перед запуском и kill-guard при <400 МБ свободных; не плодить фоновые python.
  С API-эмбеддингами ядро notebook лёгкое (нет модели) — это и спасло.
- **Дефект pgvector** воспроизводится ТОЛЬКО при `enable_seqscan=off` (форс HNSW) + фильтр,
  ортогональный запросу. На малой таблице planner идёт seq-scan → честные K (не баг).
- **recall=1.0 на чистом DBpedia** даже при ef=4 → HNSW-ef recall-размен НЕ виден; реальный
  recall-размен показываем на **квантовании BQ**. Так и оформлено в тексте честно.
- **pandas-таблицы** (`perf`, `quant`, `ef_df`) в скрипт-режиме не печатаются — добавлены
  `print(df.to_string())`; в Jupyter рендерятся как обычно (бара-выражение в конце ячейки).
- **`.ipynb` без выводов** — он собирается из `comparison.py` (jupytext). Источник истины — `.py`.
- **Кэш эмбеддингов** в `embeddings_cache/*.npy` (gitignored) — считается один раз; меняешь
  N или модель — пересчитывается. `.env` тоже gitignored (ключ не коммитим).

## 9. Как запустить (точные команды)

```bash
cd doc/materials/vector-bases/practice/comparison
docker compose up -d                      # Qdrant + pgvector
uv sync                                    # зависимости (qdrant-client 1.18, chromadb 1.5.9, pgvector 0.8.2 …)
# ключ: из канонического .env репо или свой
export OPENROUTER_API_KEY=...              # либо положить в comparison/.env
uv run python corpus.py 20000             # предсчёт эмбеддингов (один раз, ~8 мин)
# прогон как скрипт (валидация):
MPLBACKEND=Agg N_BASE=20000 N_PERF=2000,10000,20000 uv run python comparison.py
# или собрать тетрадку:
uv run jupytext --to ipynb comparison.py && uv run jupyter lab comparison.ipynb
```

Базы поднимаются в docker (порты Qdrant 6333/6334, pgvector **5433**→5432). Chroma — embedded
в процессе, сервер не нужен.
