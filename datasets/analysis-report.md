# Анализ реальных диалогов — ветка извлечения

**Дата:** 2026-06-13  
**Корпус:** 5 чатов (`datasets/dialogs/CHAT_*.json`), 57 сообщений  
**Контекст:** AI-агент llmstart.ru — первая линия B2C-продаж (понять запрос → RAG → продукт → оплата → лид)  
**Метод:** Уровень 1 — свободный разбор саб-агентом на чат → сведение оркестратором  
**Артефакты:** плоские тексты в `datasets/dialogs/flat/` (скрипт `datasets/scripts/flatten_dialog.py`)

## Рамка от эксперта (из постановки проекта)

Ожидаемые способности агента B2C:

1. Понять запрос и сегмент (B2B vs B2C)
2. Подобрать продукт из каталога через RAG (`data/b2c/`)
3. Ответить на вопросы о формате, программе, цене
4. Предложить покупку (мок-ссылка) и зафиксировать лид
5. Корректно отказать / эскалировать, если продукт не подходит

Каталог B2C (из `project-draft.md`): ai-agents-combo, vibe-coding-intensive, fullstack-aidd, agents, deep-agents, consultation.

**Ограничения корпуса:**

- Малый объём (n=5) — частоты **ориентировочные**, не статистика.
- Нет `[PHOTO]` в текущей выгрузке — медиа-контекст не терялся.
- Все чаты влезают в одно окно модели; fan-out саб-агентов применён по методике, не по необходимости размера.
- Ни один чат не дошёл до оплаты в переписке — паттерны **pre-purchase** и **отложенной конверсии** доминируют.
- Данные уже обезличены (`<AUTHOR>`, `<PERSON>`, `<URL>` и т.п.).

## Сводная таксономия групп запросов

Группы выведены из повторяющихся паттернов **между чатами**. Частота = в скольких из 5 чатов встречается (🔴 слабо / 🟡 средне / 🟢 часто).

### Группа 1. Логистика и формат обучения 🟢 (4/5 чатов)

**Что проверяем у агента:** умеет ли дать точный, структурированный ответ о delivery-модели без путаницы продуктов/годов.

| Категория | Описание | Чаты | Примеры формулировок |
|-----------|----------|------|----------------------|
| 1.1 Формат занятий | Онлайн / запись / эфиры / задания | 0014, 0110 | «какой формат у курсов?», «в каком формате проходят занятия?» |
| 1.2 Расписание и длительность | Дни, время, часы на занятие | 0014, 0020, 0110 | «расписание?», «по каким дням и во сколько?», «продолжительность?» |
| 1.3 Гибкость темпа | Свой темп vs обязательный live | 0014, 0020 | «можно в своё удобное время?», «синхрон — must» |
| 1.4 Часовой пояс / география | MSK + локальное время | 0020 (единичный) | «в эту субботу я в Сан-Франциско — во сколько?» |
| 1.5 Пробел на сайте | Информация есть, но не найдена | 0014 (единичный) | «не могу найти расписание на сайте» |

**Типичные боли:** смешение нескольких курсов/годов в одном ответе; «время пока не определили» без даты уточнения; ответ только в MSK при географической оговорке.

---

### Группа 2. Подбор продукта и product fit 🟢 (4/5 чатов)

**Что проверяем:** правильный маппинг запроса → продукт из каталога; ясность «что за предмет».

| Категория | Описание | Чаты | Примеры |
|-----------|----------|------|---------|
| 2.1 Прямой интерес к продукту | Уже знает, что хочет | 0014, 0020 | «комбо курс», «интенсив» |
| 2.2 Запрос узкой ниши | Продукт не в каталоге / неочевиден | 0070 | «курс по AI-кодингу веб-проектов (фронт/бэк)?» |
| 2.3 Состав оффера | Непонятно, что входит (комбо, линейка) | 0014, 0110 | «что за предмет преподаёте?», неявно — что такое комбо |
| 2.4 Альтернативный маппинг | Эксперт предлагает близкий курс | 0070 | «курс агентов даёт методику и для веба» |
| 2.5 Квалификация профиля | Роль, опыт, цели | 0014, 0020, 0110 | «чем занимаетесь?», «CPO, код не пишу, прототипы» |

**Типичные боли:** маппинг без объяснения *как* закрывает запрос; квалификация до закрытия информационных вопросов.

### Группа 3. Ограничения и возражения по формату 🟡 (2/5)

**Что проверяем:** распознавание hard constraints; не настаивать на неподходящем потоке.

| Категория | Описание | Чаты |
|-----------|----------|------|
| 3.1 Занятость в рабочее время | Пятница днём не подходит | 0020 |
| 3.2 Отказ от async | Запись не вариант, нужен live | 0020 |
| 3.3 Timing-барьер | «Куплю, но не сейчас» | 0070 |
| 3.4 Ожидание другого формата | 2–3 созвона в неделю вечером | 0020 (единичный) |

**Типичный исход в данных:** graceful exit → следующий поток / waitlist (без фиксации контакта).

### Группа 6. Отложенное решение и follow-up 🟢 (3/5)

**Что проверяем:** мягкий CTA после deferral; follow-up с новой ценностью.

| Категория | Описание | Чаты |
|-----------|----------|------|
| 6.1 Soft deferral | «Ознакомлюсь и решу» | 0014 |
| 6.2 Verbal commitment без оплаты | «Супер, я поучаствую» | 0070 |
| 6.3 Follow-up check-in | «Удалось ознакомиться?» | 0014 |
| 6.4 Re-engagement outbound | Оффер через месяцы | 0020, 0070 |
| 6.5 Мягкий отказ | «Уже разобрался» | 0070 |

**Типичный gap:** нет CTA (цена, ссылка, дата) после сигнала интереса; follow-up без новой ценности.
