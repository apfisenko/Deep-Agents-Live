## 4. Типы датасетов (scope v0)

Формат: **ChatML** — `input` / `expected_output` + `metadata`.

### В scope v0

| Тип | ID | Что проверяет | Объём v0 | Покрытие |
|-----|-----|---------------|----------|----------|
| **A** | `faq-format` | G1: формат, слоты, записи, TZ | ~8–12 | реал + синт |
| **B** | `product-fit` | G2: выбор SKU, комбо vs интенсив, redirect «нет такого курса» | ~10–15 | реал + синт |
| **C** | `segment-route` | G7: b2b / b2c, без B2C-оплаты для корп. | ~3–5 | реал (0127) + синт |

**Целевой размер v0:** **~22–32** single-turn (или 2-turn при необходимости).

### Отложено (не генерируем в v0)

| Тип | ID | Причина отложения |
|-----|-----|-------------------|
| D | `objection` | G4, G5 |
| E | `qualification` | G3 |
| F | `out-of-catalog` | частично закрывается типом **B** |
| G | `tool-rag` | eval tools |
| H | `tool-funnel` | оплата / lead |
| I | `rejection` | отсев клиентов |

## 5. Метрики (только A–C)

| Тип | Поля в `metadata` | Метрика |
|-----|-------------------|---------|
| A, B | `dataset_type`, `product_id`, `facts[]` | **Fact coverage** + Answer Correctness |
| C | `dataset_type`, `segment`: `b2c` \| `b2b`, `must_not`: оплата B2C для b2b | **Segment accuracy** (бинарно) + Answer Correctness |

Судья для открытых ответов — с reference `data/b2c/programs` (на этапе eval).

---

## 6. Техники

| Техника | Где |
|---------|-----|
| **Стратификация** по типу A/B/C и `product_id` | обе ветки |
| **Single-turn extraction** | извлечение |
| **Перефразирование (персоны)** | синтез |
| **SGR** | генерация записей |
| **Негативные инструкции** | не смешивать все курсы в одном ответе; C: не `create_payment_link` для b2b |
| **Обезличивание** | плейсхолдеры как в source |

---

## 7. План валидации

1. Структурная — JSONL, `dataset_type` ∈ {`faq-format`, `product-fit`, `segment-route`}.
2. Покрытие — матрица **тип × product_id** (для C — `segment`).
3. Семантическая — `facts[]` сверка с `data/b2c/programs`.
4. Validation sample ~10% глазами.

---

## 8. Ключевые решения

| Вопрос | Решение |
|--------|---------|
| Типы v0 | **Только A, B, C** (решение заказчика) |

| Длина эталона | сжатый ответ виджета (3–8 предложений) |
| Размер | ~22–32 записи |
| Файлы | `extracted.jsonl` + `synthetic.jsonl` → merge `v0.1.jsonl` |
